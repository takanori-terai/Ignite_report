# 2. イベント全体から読み解く Microsoft の戦略的メッセージ

今回の Ignite を通して Microsoft が最も強く打ち出していたテーマは、  
**“AI エージェントを前提にした企業の新しいスタンダードをつくる”** という点だったように思います。

その象徴的なキーワードとして、Keynoteでも繰り返し取り上げられていたのが  
**Frontier Firm（フロンティア企業）** という概念です。

![Frontier Firm のイメージ](画像ファイルのパスまたはURL)

キーノートの内容を各種レポートや公式発表から整理すると、Microsoft が示す 「Frontier Firm」 とは、

> **“AIと自律エージェントが業務の中心に組み込まれ、ビジネスプロセスそのものが継続的に最適化される企業”**

を指しているようで、単なる業務効率化のためのAI導入ではなく、  
**AIを一人の従業員として扱うように、組織全体をAI前提で再設計する“企業変革”が今後重要になってくる**  
というのが Microsoft が今回示した大きなメッセージです。

そして Microsoft は、この 「Frontier Firm」 の実現を支援していくために、以下の方向性を強化すると今回のカンファレンスでは述べています。

- **人や組織の“文脈”を理解したうえで主体的に動くエージェントを当たり前にすること**  
  （個々の業務状況や関係者、履歴などを理解したエージェント支援の実現）

- **アプリやシステムを横断して動くエージェント基盤を整備し、仕事の流れを自動化すること**  
  （資料作成・検索・承認・調整などが一連のエージェントでつながる世界観）

- **エージェントを安全に運用するための“型”を企業全体に提供すること**  
  （エージェントのID管理・アクセス制御・監査などの組織標準化）

これらはすべて、「人 × AIエージェント」の協働を企業の前提とするために欠かせないコンセプトとして語られており、AIを活用した新しい働き方を実現するための基盤技術や運用ノウハウを、Microsoft が一手に引き受けていくという強い意志が感じられました。

# 3. Microsoft Ignite 2025 の主要トピック

上記で述べた「Frontier Firm 実現のための 3 つの強化方向」に沿って、Ignite 2025 で特に重要だと感じた発表を整理しました。

- **Work IQ / Fabric IQ / FoundryIQ：企業の“文脈”を理解するための新しい知識レイヤー**  
- **Office 製品のエージェント対応（Copilot / MCP）：アプリ横断で仕事を自動化するエージェント基盤**  
- **Agent 365 と Entra Agent ID：エージェントを安全かつ組織的に運用するためのガバナンス基盤**

ここでは、それぞれがどの強化方向に対応しているのかを踏まえて、内容を噛み砕いて紹介していきます。

---

## 3-1. Work IQ / Fabric IQ / FoundryIQ —— “会社の文脈を理解するエージェント” をつくるための知識基盤  
**（強化方向①：人や組織の“文脈”を理解した主体的なエージェント）**

Ignite 2025 で最も印象的だったのが、Microsoft が新たに提示した **エージェントのための知能レイヤー（Work IQ / Fabric IQ / FoundryIQ）** です。これは、企業のあらゆる情報を「AI エージェントが理解できる形」に整えるための基盤で、いわば **“会社専用エージェントの頭脳”** のようなものだと考えています。

### ● Work IQ：人・組織・仕事の流れを理解する  

![Work IQ のイメージ](画像ファイルのパスまたはURL)


メール・Teams・会議メモ・ファイル利用履歴などから、

- いま誰が何をしているか  
- どの資料がどの会議と関係しているか  
- 過去にどんな判断や議論があったか  

といった **“働き方の文脈”** を AI が理解できるようにしてくれるのがWork IQのようです。

この機能をエージェントに持たせることによって、たとえば「来週の会議資料を更新して」と依頼すると、関連資料の履歴や関係者を踏まえた上で、次に必要な作業を見つけて提案するといった“気の利く動き”が可能になります。

### ● Fabric IQ：ビジネスデータを意味ごと理解する  

![Fabric IQ のイメージ](画像ファイルのパスまたはURL)

売上・在庫・顧客情報など、企業が持つ膨大なデータを **“ビジネスの意味づけ付き”** で扱えるようにするレイヤーです。

これにより、

- 「直近 3 か月の売上をまとめて」  
- 「顧客別に離脱リスクを分析して」

といった自然言語の指示に対して、エージェントが文脈に沿ってデータを取り出し、分析やレポートを自動生成できるようになります。

### ● FoundryIQ：会社固有の知識を統合し、エージェントに安全に渡す  

![FoundryIQ のイメージ](画像ファイルのパスまたはURL)

FoundryIQは、上記のWork IQ と Fabric IQ を統合し、加えて社内システムや外部情報まで含めた  
**“会社独自の知識ベース”** を構築してくれます。

従来の RAG（検索＋生成）では情報整備や運用が属人化しがちでしたが、FoundryIQ によって

- 情報の正確性  
- ガバナンス（アクセス管理）  
- セキュリティ  

を維持したままエージェントに知識を提供できるようになるようです。

> **まとめると**：  
> ・**Work IQ** が「いま何が起きているか」を、  
> ・**Fabric IQ** が「正しいデータの意味」を、  
> ・**FoundryIQ** が「企業が持つ知識の扱い方」を、  
> エージェントに渡すことで、Copilotなどのエージェントはより文脈を理解した高度な支援が可能になります。

## 3-2. Office 製品（PowerPoint / Word / Excel）のエージェント対応  
**（強化方向②：アプリやシステムを横断して自動化するエージェント基盤）**

![Office 製品のエージェント対応イメージ](画像ファイルのパスまたはURL)

2つ目に注目すべきポイントは、Word・PowerPoint・Excel といった **Office 製品そのものが「MCP対応」「エージェント化」したこと** です。

これにより、Microsoft 365 Copilot や外部のエージェントが Office の機能を直接呼び出し、アプリ横断の業務を自動化できるようになりました。

### ● 自然言語だけで資料作成や分析が完結  
「この会議メモから PowerPoint を作って」  
「この Excel の売上表を分析して」  
「提案書の第 3 章だけ、過去資料を参考に書き直して」

といった依頼に応じて、Copilot や接続したエージェントが該当アプリを操作し、必要なアウトプットを生成します。

たとえば、

- CRM に入力されたデータから **定例営業資料を PowerPoint で自動更新**  
- 承認ワークフローの最終段階で **申請サマリー PDF を自動生成**  
- Slack の指示で **Word の企画書ドラフトを自動作成**  

といった、**アプリ横断の“仕事の流れ”そのものをエージェントが担う世界**が現実味を帯びてきました。

> **まとめると**：  
> これまで人が手作業で行っていた資料作成・集計・分析・調整といった業務が、エージェントによって一連のプロセスとして自動化できるようになります。

---

## 3-3. Agent 365 と Entra Agent ID —— “安全に使えるエージェント” の組織標準化  
**（強化方向③：エージェントを企業として安全に運用するための“型”の提供）**

3つ目の大きな発表が、**Agent 365** と **Entra Agent ID** です。  
これは、企業が大量のエージェントを安全に扱うための“ガバナンス基盤”として位置付けられています。

### ● Agent 365：エージェント運用の司令塔  

![Agent 365 のイメージ](画像ファイルのパスまたはURL)

Agent 365 は、社内のエージェントをまとめて管理する **コントロールプレーン** のようなもので、

- そのエージェントは何を実行したか  
- どのデータにアクセスしたか  
- どのユーザーが利用したか  
- 異常な動作をしていないか  

などを、**人間のユーザー管理と同じ粒度で監視・管理**できます。

これは、エージェントが企業内に大量に導入されていくフェーズでは必須の機能です。（誰が作ったかわからないエージェントをいきなり社内に導入するのはセキュリティ的にも怖いですからね。）

### ● Entra Agent ID：エージェント向けの正式な“IDアカウント”  

![Entra Agent ID のイメージ](画像ファイルのパスまたはURL)

さらに、これまでは曖昧だったエージェントの身元を、  
**Entra ID（旧 Azure AD）で正式なアカウントとして登録できる**ようになりました。

これにより、エージェントに対しても、
- アクセス権限の付与・剥奪  
- 認証  
- ログ管理  
- コンプライアンス対応  
を **人間に対して行っていたのと同じ仕組み** で運用できるようになります。

たとえば、  
「請求書データにアクセスできるのは経理エージェントだけ」  
「営業データにアクセスできるのは CRM エージェントのみ」  
といった **役割ベースのアクセス管理（RBAC）** を安全に設定できます。

> **まとめると**：     
> エージェントがどれだけ増えて複雑化したとしても「安全に・統制の取れた形で」運用できるようにするための土台が整備された、と言えそうです。

# 4. NTT DATA ブースの展示模様
ここまで Ignite 2025 の主要トピックを Microsoft の戦略的メッセージに沿って整理してきましたが、最後に、私が担当した NTT DATA ブースでの展示内容についても簡単に紹介しておきます。

今回の NTT DATA ブースでは、日本・アメリカ・欧州・インドをはじめ、各リージョンが取り組む多様なプロジェクトを紹介しました。

![NTT DATA ブースの様子](画像ファイルのパスまたはURL)

環境保護区域の来訪者向けの AI を活用したチャットボットソリューション（デンマーク）、非エンジニアがAIを簡単に構築・管理するためのサービス（インド）など、地域特性と課題に応じた幅広い事例が並び、来場者からも高い関心が寄せられていました。

その中の一部として私は、**エンタープライズグレードの AI エージェントを業務システムに組み込むためには何が必要か**という観点から、**EC サイトを自律的に操作するエージェントのライブデモ**を通じて説明を行ってきました。（デモの構築にはMicrosoft が最近発表した最新フレームワークである **Microsoft Agent Framework** を活用しました）

![Retailデモのイメージ](画像ファイルのパスまたはURL)

来場者との議論では、

- 実務に適用する際、どこまでエージェントに任せて良いのか  
- どのポイントに制御や監査の仕組みを設けるべきか  
- 人とシステムの責任範囲をどのように設計すべきか  

といった、SIer だからこそ提供できる実装・運用の視点で深い対話が生まれ、NTT DATA が有するエンタープライズ領域での知見や強みを、確かな手応えをもって来場者の方に伝えられたと感じています。
